Motivation

From a question to a campaign — the gap we close

why a multi-agent framework

A scientist can ask “how is recharge vs. runoff partitioned across this watershed?” in seconds. Running it takes days: pick sites, fetch terrain/soil/observations, configure dozens of single-column models, build, run, validate.

Domain knowledge

Which HUC? How many columns? What forcing, soil, validation data?

Real data, not guesses

Coordinates, soil profiles and water tables must come from authoritative datasets — never an LLM’s imagination.

HPC execution

CIME case creation, builds, SLURM, per-column configuration at scale.

The framework automates all three while keeping the scientist in the loop.

Core idea

Three tiers — the LLM reasons, Python materializes, ELM executes

separation of concerns & anti-hallucination

Tier 1 · LLM reasoning

Reception frames the question and gathers evidence via MCP tools; the planner designs the experiment strategy.

Never emits coordinates.

Tier 2 · Expander

Deterministic Python turns the strategy into concrete column points from the real DEM, soil and water-table data.

Tier 3 · Execution

Strict per-column ELM configuration, CIME build/clone, and SLURM execution → history files.

Key invariant: every latitude/longitude is materialized from data in Tier 2, so the LLM cannot hallucinate a location.

Tier 1 · agent

Reception — an agentic, pure-LLM tool user

request → framed brief

What it does

  • Classifies intent & archetype: conceptual · site · coupling
  • Drives the MCP tools itself in a loop (no hard-coded domain logic)
  • Resolves the watershed, samples terrain, pulls soil / wells / WTD
  • Emits a compact JSON brief for the planner

Human-in-the-loop

  • On material ambiguity (which American River? what HUC scale?) it can ask the user — backed by MCP candidates
  • ask_user tool, exposed only in interactive mode
  • Falls back to best-judgment + recorded assumption in batch mode

Generic runtime: ToolLoopAgent exposes MCP tools as OpenAI functions and passes tools= every round (Bedrock/Claude requirement).

Multi-agent LLM framework8
Tier 2 · expander

Deterministic expansion — strategy → concrete columns

no LLM, no invented coordinates

1 · Sample

Sample the real DEM across the basin bbox (terrain/3DEP), clip to the watershed polygon.

2 · Stratify

Bin points into elevation bands; allocate the planner’s N proportional to occupied area (≥1 per band).

3 · Enrich

Farthest-point spread within each band; attach SSURGO soil profile + Fan equilibrium water-table depth per point.

Output: columns.json — every column carries a real lat/lon, elevation band, soil profile (van Genuchten), and water-table depth.

Multi-agent LLM framework9
Tier 3 · execution

Per-column ELM execution on Perlmutter

strict configuration · fast cloning · SLURM

Configuration bridge

  • columns_to_elm_plan → one coupler per column, each with its own lat/lon + soil
  • Generates per-column surface + domain NetCDF

Build & run

  • CIME create_newcase / case.setup / build once
  • Clone columns with --keepexe (~15 s each, shared exe)
  • srun per column on an salloc node → history files

Repairing the --keepexe clone path (EXEROOT-resolved exe + case.setup --reset) cut per-column cost from a ~5-min rebuild to ~15 s.
