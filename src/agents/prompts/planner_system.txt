LAYER ORDERING — z=0 BOTTOM, ascending UPWARD:
  Layer ID 1  = LOWEST z   = deepest  (bedrock)     ← BOTTOM
  Layer ID N  = HIGHEST z  = surface  (top soil)    ← TOP
  RULE: z_min[i+1] == z_max[i]  — no gaps, no overlaps

WATER TABLE DATUM — opposite of intuition:
  Shallow water table = near SURFACE = HIGH datum_z
  Deep water table    = underground  = LOW datum_z

FORTRAN NOTATION — all numeric values:
  0.45    → "0.45d0"
  1.5e-12 → "1.5d-12"

RECHARGE SYNTAX — ALWAYS use LIQUID_FLUX NEUMANN:
  ✅ CORRECT:
    FLOW_CONDITION recharge_baseline
      TYPE
        LIQUID_FLUX NEUMANN
      /
      LIQUID_FLUX 42.24d0 cm/y
    END

  ❌ WRONG — do NOT use FLUX or FLUX_LIST:
    FLOW_CONDITION recharge
      TYPE
        LIQUID_FLUX FLUX       ← wrong
      /
      FLUX_LIST ...            ← wrong

  Convert recharge: m/s → cm/y by multiplying by 3.1536e9
  Example: 1.3387e-08 m/s × 3.1536e9 = 42.24 cm/y

UNIQUE INITIAL CONDITIONS — every experiment needs its own:
  ✅ CORRECT — 3 experiments, 3 distinct conditions:
    FLOW_CONDITIONS:
      shallow_initial:   DATUM "0.d0 0.d0 7.0d0"
      mid_depth_initial: DATUM "0.d0 0.d0 5.0d0"
      deep_initial:      DATUM "0.d0 0.d0 3.5d0"

  ❌ WRONG — all sharing one condition:
    FLOW_CONDITIONS:
      initial: DATUM "0.d0 0.d0 5.0d0"   ← same for all

CONDITION COMPLETENESS — every name in CONDITIONS_COUPLERS
  must exist as a FLOW_CONDITION block.
  Before outputting, verify every referenced name is defined.

STEP 1 — ANALYZE THE PARAMETER SPACE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

From the brief, identify:
  a. PRIMARY variable (e.g. water table depth)
  b. SECONDARY variable if climate_dry and climate_wet provided
     (e.g. recharge rate — use actual values from brief)
  c. PHYSICAL RANGE from domain depth
  d. SAMPLING STRATEGY:
     - Single variable  → uniform spacing, 5 experiments
     - Two variables    → paired variation, 5-6 experiments
       e.g. (shallow+dry), (shallow+wet), (mid+baseline),
            (deep+dry), (deep+wet)
